# Agent影视助手命令总览

这份文档只做一件事：把当前主线命令讲清楚。

适用范围：

- MoviePilot 插件 API
- Agent影视助手内置飞书入口
- 外部智能体通过 `agent-resource-officer` skill / helper 调用

不同入口会有少量别名差异，但主命令尽量保持一致。

---

## 先记住 4 条

1. `搜索 <片名>` 默认优先走 MP/PT；如果 MP/PT 已关闭，再按当前启用源回退。
2. 标题级 `转存 <片名>` / `115转存 <片名>` / `夸克转存 <片名>` / `云盘搜索 <片名>` 已取消。现在统一建议先搜索，再从结果里回编号继续处理。
3. `下载 <片名>` 走 MoviePilot 原生 PT 下载链；如果你说 `最新集` / `第3集`，会先按目标集数筛结果或方案。
4. 在 PT 结果里，`1` / `选择 1` / `下载1` 都是直接下载；在云盘/影巢结果里，`1` / `选择 1` 是继续处理第 1 条。
5. `搜索 第 3 集`、`搜索 E03`、`搜索 最新集` 这类带集数线索的写法，会直接按 MP/PT 搜索。
6. `检查 大君夫人`、`检查大君夫人` 这类粘连写法，会按更新检查处理；`检查115登录` 仍然是登录检查。
7. 云盘侧更新检查仍然保留来源前缀：`盘搜更新检查 <片名>` / `影巢更新检查 <片名>`。

## 搜索类

| 命令 | 作用 |
|---|---|
| `MP搜索 <片名>` | 走 MoviePilot 原生 PT 搜索 |
| `PT搜索 <片名>` | 同 `MP搜索` |
| `原生搜索 <片名>` | 同 `MP搜索` |
| `搜索 <片名>` | 默认优先走 MP/PT；如果 MP/PT 已关闭，再按当前启用源回退 |
| `盘搜搜索 <片名>` | 先查盘搜；盘搜没结果时按开关补查影巢 |
| `影巢搜索 <片名>` | 先查影巢；影巢没结果时按开关补查盘搜 |
| `下载 <片名>` | 先识别媒体，再走 MoviePilot 原生 PT 下载链 |
| `更新检查 <片名>` | 检查当前媒体是否有更新资源 |
| `检查 <片名>` | 同 `更新检查` |
| `盘搜更新检查 <片名>` | 只看盘搜侧更新资源 |
| `影巢更新检查 <片名>` | 只看影巢侧更新资源 |

补充说明：

- `MP搜索` / `PT搜索` 遇到片名歧义时，通常会先让你选正确影片或剧集。
- `下载 <片名>` 如果片名有歧义，也会先让你选影片，再继续 PT 下载链。
- `搜索 第4集` / `搜索 最新集` 会优先把目标集相关结果排到前面。

## 选择、详情、翻页

| 命令 | 作用 |
|---|---|
| `1` | PT 结果里直接下载；云盘结果里继续处理第 1 条 |
| `选择 1` | 同 `1` |
| `选择 1 详情` | 仅云盘/影巢：查看第 1 条详情 |
| `详情 1` | 仅云盘/影巢：查看第 1 条详情 |
| `下载1` | PT 结果里直接下载第 1 条 |
| `n` | 下一页 |
| `下一页` | 同 `n` |

这里最容易混淆的是 `1`：

- 在 PT 结果里，`1` 会直接下载第 1 条
- 在云盘结果里，`1` 通常是继续第 1 条，并执行对应选择逻辑

PT 结果不提供详情卡；云盘/影巢如果只想看详情，用 `选择 1 详情` 或 `详情 1`。

## PT 下载类

| 命令 | 作用 |
|---|---|
| `下载 <片名>` | 先找片，再生成 3 个下载方案 |
| `下载 <片名> 最新集` | 优先按最新集生成 3 个方案 |
| `下载 <片名> 第3集` | 优先按指定集生成 3 个方案 |
| `1` | 在 PT 结果里直接下载当前第 1 条 |
| `下载1` | 同 `1` |

PT 下载主线现在更简单：

```text
下载 流浪地球2
1
```

或者：

```text
MP搜索 流浪地球2
1
```

不再建议依赖一长串计划类口令。

## 飞书入口的额外说明

飞书入口除了上面这些主命令外，还有一些历史兼容别名。

新手建议：

1. 直接使用完整命令，不要依赖缩写。
2. 尤其优先用 `MP搜索`、`盘搜搜索`、`影巢搜索`、`下载`、`更新检查` 这些主命令。

如果你只想最稳地用飞书，就把它当成一个命令聊天窗口，不要自己发明新句式。
